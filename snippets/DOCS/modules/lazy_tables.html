Module lazy_tables

This module provides utilities for tables whose members are generated lazily.

Functions

MakeOnDemand (make, is_nullary)
Builds a new table. If one of the table's keys is missing, it will be filled in automatically when indexed with a new object.

The table's metatable is fixed.

Parameters:

  • make: callable Routine called when a key is missing, that may take the key as argument and returns a value to be assigned.
  • is_nullary: bool make receives no argument?

Returns:

    table Table.

MemberTable (object, name)
Gets an object's member. If the member does not exist, a new table is first created and assigned as the member.

Note that if the member already exists it may not be a table.

Parameters:

  • object: Object to query.
  • name: Member name.

Returns:

    table Member.
SubTablesOnDemand (choice, weakness, cache)
Builds a new table. If one of the table's keys is missing, it will be filled in automatically with a subtable when indexed.

Note that this effect is not propagated to the subtables.

The table's metatable is fixed.

When called in a bound table context, the binding is used as the destination table.

Parameters:

  • choice: string If nil, subtables will be normal tables.

    Otherwise, the weak option, as per table_ops.Weak, to assign a new subtable.

  • weakness: string The weak option, as per table_ops.Weak, to apply to the table itself.

    If nil, it will be a normal table.

  • cache: callable Optional cache from which to pull subtables.

    If nil, fresh tables will always be supplied.

Returns:

    table Table.
