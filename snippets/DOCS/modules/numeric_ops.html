ClampIn (n, minb, maxb)
Clamps a number between two bounds.

The bounds are swapped if out of order.

Parameters:

  • n: number Number to clamp.
  • minb: number Minimum bound.
  • maxb: number Maximum bound.

Returns:

    number Clamped number.

MinMax_Interval (a, b, m, n)
Variant of MinMax that contrains its arguments to an interval.

N.B. As a side effect of clamping, if both a and b are < m, or both > n, the "maximum" will be < m, or the "minimum" > n, respectively, i.e. the return values will be out of order.

Parameters:

  • a: number Value #1.
  • b: number Value #2.
  • m: number Range lower bound.
  • n: number Range upper bound, ≥ m.

Returns:

  1. number Clamped minimum value, ∈ [m, +∞).
  2. number Clamped maximum value, ∈ (-∞, n].
MinMax_Range (a, b, n)
Variant of MinMax that contrains its arguments to a range.

The return values proviso applies as per MinMax_Interval.

Parameters:

  • a: number Value #1.
  • b: number Value #2.
  • n: number Range limit ≥ 1.

Returns:

  1. number Clamped minimum value, ∈ [1, +∞).
  2. number Clamped maximum value, ∈ (-∞, n].
